System Login Module

50% of the module must be working and submitted to show progress before the final output.


User Interface (UI):
Basic login form with username/email and password fields.

Buttons for login, reset/clear, and exit.

Initial error messages for empty fields.

Basic Functionality:
Input validation (cannot leave fields blank).

Password masking (e.g., “- - - - - ”).

Initial connection to the database for checking hardcoded or test credentials.

Set length of characters.

Partial Database Integration:
A simple user table in the database with test accounts.

Verification of correct vs. incorrect login inputs.

Basic Navigation:
Successful login redirects to a user dashboard.

Failed login shows an error message.
